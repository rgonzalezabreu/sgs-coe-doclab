  2. Start project This command generates a default project named SGS.DocfxSite. docfx init -q --output SGS.DocfxSite

Commands

  • docfx init -q --output SGS.DocfxSite
    • Create project
  • docfx .\SGS.DocfxSite\docfx.json --serve
    • Build and serve doc site
  • docfx .\SGS.DocfxSite\docfx.json --build
    • Build documentation
  • docfx serve .\SGS.DocfxSite\_site\
    • Serve doc site
  • docfx template export default
    • Export template

Template installation

We are using Material custom template.

All templates are available in https://dotnet.github.io/docfx/templates-and-plugins/templates-dashboard.html

Download template fron https://github.com/ovasquez/docfx-material and places in .\SGS.DocfxSite\templates\material and then reference in docfx.json

...
"template": [
    "default",
    "templates/material"
]
...

Code project files

All code is required to copy on .\SGS.DocfxSite\src\ to process on build pipeline, create folders for each project is required.

.\api-blod -> SGS Framework Azure BlobStorage
.\api-funk -> SGS Framework FunK

Specify content metadata and build files:

...
"metadata": [
    {
      "src": [
        {
          "files": [
            "src/SGS.Framework.Azure.SimpleBlobStorage/*.csproj"
          ]
        }
      ],
      "dest": "api-blob"
    },
    {
      ...
    }
  ],
  "build": {
    "content": [
      {
        "files": [
          "api-blob/**.yml",
          "api-blob/index.md"
        ]
      },
      ...
    ]
  }
  ...

Readme project files

All project Readme files can be copied on api folder documentation, but if the project not have a Readme file you can put index instead.

.\api-blod
  - index.md
